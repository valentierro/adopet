# Glossário

Termos técnicos e de domínio usados no Adopet.

## Domínio

| Termo | Significado |
|-------|-------------|
| **Tutor** | Dono do anúncio — pessoa que está doando o pet |
| **Adotante** | Pessoa interessada em adotar o pet |
| **Pet** | Animal anunciado (cachorro ou gato) |
| **Swipe** | Ação de curtir (like) ou passar (pass) no feed |
| **KYC** | Verificação de identidade (documento + selfie) |
| **ONG** | Parceiro tipo organização não governamental |
| **Parceiro** | ONG, clínica ou loja cadastrada no app |

## Tecnologias

| Termo | Significado |
|-------|-------------|
| **Monorepo** | Vários projetos em um único repositório |
| **pnpm** | Gerenciador de pacotes (alternativa ao npm) |
| **NestJS** | Framework Node.js para APIs |
| **Prisma** | ORM para acessar o banco |
| **Expo** | Ferramenta para desenvolver apps React Native |
| **Zustand** | Biblioteca de estado para React |
| **React Query** | Cache e fetch de dados da API |
| **JWT** | JSON Web Token — token de autenticação |
| **EAS** | Expo Application Services — build e deploy |

## Código

| Termo | Significado |
|-------|-------------|
| **Hydrate** | Carregar estado persistido (ex: tokens do storage) |
| **Seed** | Dados iniciais inseridos no banco |
| **Migration** | Alteração no schema do banco (versionada) |
| **Presign** | URL temporária para upload direto no S3 |
| **Cursor** | Token para paginação (próxima página) |
